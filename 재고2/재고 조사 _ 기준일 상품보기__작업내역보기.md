# 페이지 분석 — 작업내역보기

## 1. 페이지 개요

| 항목 | 내용 |
|------|------|
| 메뉴 그룹/위치 | 재고 조사 > 기준일 상품보기 |
| 페이지명 | 작업내역보기 |
| 식별 코드 | 미기재 |
| 목적 | 재고조사 진행 중 특정 상품을 조회하여 기준일 재고와 실산수량을 비교·기록하는 메뉴. 담당자가 현장에서 실물을 확인하며 실산내역(실제 카운트한 수량)을 직접 입력·저장하고, 입력 이력을 추적·수정할 수 있다 |
| 접근 권한 | 물류사 작업자, 물류사 관리자, 시스템 관리자 |

> **활용 시나리오**: 재고조사 현장에서 담당자가 상품 바코드를 스캔하거나 상품코드를 입력해 해당 상품의 기준일 전산재고를 즉시 확인하고, 실물 카운트 결과(실산수량)를 입력하여 저장한다. 저장된 실산내역은 가입고 현재고 대조(R-STK-605) 및 재고조정(R-STK-606)의 근거 자료로 활용된다.

---

## 2. 업무 흐름 (Business Flow)

```
[기준일 상품보기] (R-STK-607) 에서 [작업내역보기] 탭/메뉴 진입
    ↓
화주사 · 창고 · 기준일 · 조사차수 설정
    ↓
상품 검색 (바코드 스캔 또는 상품코드/스타일코드 입력)
    ↓
상품 기준일 재고 정보 표시
    │── 기준일 재고수량 (전산 스냅샷)
    │── 가입고 누계 수량
    │── 기 작성된 실산내역 (있는 경우)
    ↓
실산수량 직접 입력 → [저장]
    ↓
실산내역 목록에 기록 (수정/삭제 가능)
    ↓
결과 확인 → 엑셀 다운로드 or 재고조정 요청(R-STK-606)으로 이동
```

## 3. 화면 구성

### 3-1. 검색 조건 (헤더 영역)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 화주사 | Select | ✅ | 조회 대상 화주사 |
| 창고 | Select | ✅ | 조회 대상 창고 |
| 기준일 | Date | ✅ | 재고 스냅샷 기준일. 기본값: 최신 차수의 기준일 |
| 조사 차수 | Select | - | 특정 재고조사 회차 선택 |
| 상품코드 | Input | - | 내부 SKU 코드 직접 입력 |
| 바코드 | Input (스캔 가능) | - | EAN-13 바코드 스캔 또는 입력 |
| 스타일코드 | Input | - | 스타일 단위 조회 |
| 브랜드 | Select (다중) | - | 브랜드 필터 |
| 색상 | Select (다중) | - | 색상 필터 |
| 사이즈 | Select (다중) | - | 사이즈 필터 |
| 실산 상태 | Select | - | 전체 / 미입력 / 입력완료 / 수정됨 |

**[검색] [초기화] 버튼**

## 4. 실산 상태값 정의

| 상태 | 설명 | 색상 |
|------|------|------|
| 미입력 | 해당 SKU에 실산수량이 한 번도 입력되지 않음 | Gray |
| 입력완료 | 실산수량이 저장된 상태 | Green |
| 수정됨 | 최초 저장 후 수정이 발생한 상태 (수정 이력 추적) | Orange |

## 5. 실산내역 수정·삭제 규칙

| 상황 | 처리 방식 |
|------|----------|
| 실산수량 수정 | 최초 값과 수정 값·수정 일시·수정자 이력 보존 (감사 로그) |
| 실산내역 삭제 | 관리자 이상만 삭제 가능. 삭제 사유 입력 필수. 소프트 삭제(논리 삭제)로 처리 |
| 이미 재고조정 요청에 연계된 실산내역 수정 | 수정 불가. \"해당 실산내역은 재고조정 요청(R-STK-606)에 연계되어 있어 수정할 수 없습니다.\" 안내 |
| 차수 마감 후 수정 시도 | 관리자 이상만 수정 가능. 일반 작업자는 잠금 상태 |

## 6. 입력 검증 규칙

| 상황 | 처리 방식 |
|------|----------|
| 실산수량 음수 입력 | \"실산수량은 0 이상이어야 합니다.\" 검증 에러 |
| 미등록 바코드 스캔 | \"등록되지 않은 바코드입니다: {barcode}\" 토스트 |
| 실산수량 미입력 상태로 저장 시도 | \"실산수량을 입력하세요.\" 필수 검증 |
| 기준일 재고 대비 실산수량 30% 초과 차이 | \"기준일 재고 대비 실산수량 차이가 큽니다 ({차이율}%). 계속 저장하시겠습니까?\" 확인 팝업 |
| 동일 SKU 중복 입력 | 기존 실산내역 존재 안내 + 덮어쓰기 여부 확인 팝업 |

## 8. API/이벤트

| API | Method | 설명 |
|-----|--------|------|
| `/api/stock/inventory-snapshot/product-detail` | GET | 상품코드/바코드 기반 기준일 재고 정보 조회 |
| `/api/stock/physical-count/list` | GET | 실산내역 목록 조회 (필터 파라미터 포함) |

| `/api/stock/physical-count/{id}/delete` | DELETE | 실산내역 삭제 (관리자) |
| `/api/stock/physical-count/excel-download` | GET | 실산내역 엑셀 다운로드 |

## 10. 예외 처리 및 알림

| 상황 | 처리 방식 |
|------|----------|
| 조회 결과 없음 | \"조회 조건에 해당하는 상품이 없습니다.\" 안내 |
| 실산내역 저장 중 네트워크 오류 | 자동 재시도(3회). 실패 시 \"저장에 실패했습니다. 인터넷 연결을 확인하세요.\" 에러 |
| 차수 마감 상태에서 작업자 입력 시도 | \"해당 조사 차수는 마감되었습니다. 관리자에게 문의하세요.\" 안내 |
| 대용량 엑셀 다운로드 (5,000건 초과) | 비동기 처리 후 완료 시 인앱 알림 |
