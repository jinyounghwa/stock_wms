# 페이지 분석 — 재고조사

## 1. 페이지 개요

| 항목 | 내용 |
|------|------|
| 메뉴 그룹/위치 | 재고관리 > 재고조사 |
| 페이지명 | 재고조사 |
| 목적 | 창고 내 실물 재고를 바코드 스캔·수동 입력·엑셀 업로드로 조사하고, 전산 재고와 대조하여 차이를 확인·조정하는 통합 메뉴. 작업자 화면과 관리자 화면을 분리하여 제공하며, 재고조정 승인 워크플로우를 하나의 메뉴에서 완결한다 |
| 탭 구성 | 재고 조회 / 조사 재고 입력 / 현재고 대조 / 작업내역 / 재고 조정 |
| 용어 변경 | 기존 **가입고** → **조사 재고** 로 전면 변경 |
| 접근 권한 | 물류사 작업자, 물류사 관리자, 시스템 관리자 |

---

## 2. 전체 업무 흐름 (Business Flow)

```
[조사 시작]
    ↓
[탭1: 재고 조회]
화주사 선택 + 바코드/위치 스캔 → 전산재고 즉시 확인
    ↓
[탭2: 조사 재고 입력]
실물 카운트 결과 입력
  ├── 개별 입력 (바코드 수동)
  ├── 바코드 스캔 (HT/카메라)
  └── 엑셀/CSV 업로드 (일괄)
    ↓
[탭3: 현재고 대조]
조사 재고 vs 전산 현재고 비교 → 차이 확인
    ↓
[탭4: 작업내역]
차수별 등록 현황 / 상품별 조회 / 미처리 추적
    ↓
[탭5: 재고 조정]
  작업자: 위치 스캔 → 실제 수량 입력 → 조정 요청
  관리자: 요청 목록 확인 → 승인 / 반려
    ↓
[조사 완료]
```

## 3. 탭1 — 재고 조회

### 3-1. 화면 구성

**검색 영역**

| 필드명 | 타입 | 설명 |
|--------|------|------|
| 화주사 | Select | 화주사 선택 (필수) |
| 바코드 스캔 또는 검색어 입력 | Input + 스캔 버튼 | 상품 바코드(EAN-13), 로케이션 바코드, 상품명 텍스트 검색 |

**초기 화면**: 바코드 스캔 아이콘 + "바코드를 스캔하거나 검색어를 입력해 주세요" 안내 표시

---

**결과 리스트 — 상품 단위 그룹**

| 항목 | 설명 |
|------|------|
| 상품 이미지 | 썸네일 (없을 시 기본 이미지) |
| 상품명 | 상품 풀네임 |
| 옵션 | 색상/사이즈 표기 |
| 바코드 번호 | EAN-13 |
| 브랜드명 | 링크 형태 표시 |
| 상품 수 / 로케이션 수 | 전체 수량 합계 + 적재 위치 수 |
| 로케이션 목록 | 존 구분 태그(입고존/보관존/출고존) + 위치코드 + 수량 |
| 로케이션 N개 더보기/접기 | 2개 초과 시 더보기 표시, 클릭 시 전체 펼침 |

---

### 3-2. 액션 버튼

| 버튼 | 동작 |
|------|------|
| 재고 조회 이동 | 탭1 진입 |
| 조사 재고 입력 이동 | 탭2로 이동하여 해당 상품 입력 즉시 시작 |

## 4. 탭2 — 조사 재고 입력

> 기존 **상품 가입고** 메뉴와 통합. 용어 **가입고 → 조사 재고**로 전면 변경

### 4-1. 입력 방식 선택 (서브탭)

| 서브탭 | 설명 |
|--------|------|
| 개별 입력 | 바코드 직접 입력 + 로케이션 + 수량 한 건씩 추가 |
| 바코드 스캔 | 스캔 필드 항시 포커스, 스캔 즉시 목록 누산 |
| 엑셀/CSV 업로드 | 파일 업로드 후 검증 → 일괄 등록 |
| 조사 재고 목록 | 등록된 조사 재고 차수별 조회 + 실입고 처리 |

---

### 4-2. 개별 입력

**입력 폼**

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 화주사 | Select | ✅ | |
| 조사 차수 | Select | ✅ | 기본값: 진행 중 차수 |
| 바코드 | Input | ✅ | EAN-13, Tab으로 다음 필드 이동 |
| 로케이션 | Input | ✅ | 위치 코드 입력 또는 스캔 |
| 수량 | Number | ✅ | Enter 입력 시 목록 추가 |
| 메모 | Input | - | 특이사항 자유 입력 |

- 임시 목록에 누산 → **[전체 저장]** 버튼으로 일괄 저장
- 동일 바코드+로케이션 중복 시: "기존 항목에 수량을 합산하시겠습니까?" 확인 팝업
- 화면 이탈 시 미저장 항목 존재하면 경고 팝업

---

### 4-3. 바코드 스캔

| 항목 | 설명 |
|------|------|
| 스캔 필드 | 항시 포커스 유지 (자동 재포커스) |
| 스캔 모드 | **자동 누산** (동일 바코드 스캔 시 +1) / **단위 입력** (스캔마다 수량 입력 팝업) |
| 성공 피드백 | 초록 플래시 + 상품명 표시 |
| 실패 피드백 | 빨간 플래시 + "미등록 바코드" 경고 |
| 자동 저장 | 100건 누산 시 자동 임시 저장 |
| 로케이션 설정 | 스캔 시작 전 로케이션 선택/스캔 필수 |

---

### 4-4. 엑셀/CSV 업로드

**업로드 템플릿 컬럼**

| 컬럼 | 필수 | 설명 |
|------|------|------|
| 바코드 | ✅ | EAN-13 |
| 로케이션 | ✅ | 위치 코드 |
| 수량 | ✅ | 양의 정수 |
| 메모 | - | 선택 입력 |

- 드래그 앤 드롭 업로드 지원 (.xlsx, .csv)
- 업로드 후 검증 프리뷰 표시 (오류 행 빨간색 하이라이트)
- **부분 등록**: 유효 행만 등록하고 오류 행 스킵 가능
- 오류 리포트 다운로드 버튼 제공
- 업로드 이력: 차수별엑셀 업로드 리스트에서 관리 (파일명, 총행수, 성공/실패 건수, 처리일시, 90일 원본 파일 재다운로드)

## 5. 탭3 — 현재고 대조

> 기존 **가입고 현재고 대조 (R-STK-605)** 통합

### 5-1. 화면 구성

**필터**

| 필드명 | 타입 | 설명 |
|--------|------|------|
| 화주사 | Select | |
| 창고 | Select | |
| 조사 차수 | Select | |
| 기간 | DateRange | |
| 대조 결과 | Select | 전체 / 일치 / 불일치+ / 불일치- / 미처리 |
| 최소 차이 수량 | Number | N 이상 차이 항목만 표시 |

**요약 집계 카드**

| 항목 | 설명 |
|------|------|

| 불일치(+) 건수 | 조사재고 > 현재고 |
| 불일치(-) 건수 | 조사재고 < 현재고 |
| 미처리 건수 | 실입고 미완료 항목 |

**대조 결과 목록 컬럼**

| 컬럼명 | 설명 |
|--------|------|
| 화주사 | 화주사명 |
| 바코드 | EAN-13 |
| 상품명 / 옵션 | 상품명 + 색상/사이즈 |
| 로케이션 | 위치 코드 |
| 조사 수량 | 조사 재고 입력 수량 |
| 현재고 | 전산 현재고 수량 |
| 차이 수량 | 조사수량 - 현재고 (양수 파란색 / 음수 빨간색) |
| 대조 결과 | 일치(Green) / 불일치+(Blue) / 불일치-(Red) / 미처리(Gray) |

**액션 버튼**

| 버튼 | 권한 | 동작 |
|------|------|------|
| 선택 재고조정 요청 | 관리자 | 선택 항목 재고조정 탭으로 연계 |

## 7. 탭5 — 재고 조정

> 기존 앱(APP) 화면 기반 WEB 버전으로 구현. **작업자 화면**과 **관리자 화면** 분리 제공

### 7-1. 작업자 화면 — 조정하기

**Step 1: 위치 선택**

| 항목 | 설명 |
|------|------|
| 화주사 선택 | Select (필수) |
| 구역/위치 스캔 또는 검색 | 위치 바코드 스캔 또는 위치 코드 텍스트 입력 |

**Step 2: 위치 내 상품 목록**

위치 스캔/입력 후 표시:

| 항목 | 설명 |
|------|------|
| 존 태그 + 위치코드 | 예: 보관존 4E-A-01-1 |
| 상품 종수 | 예: 상품 5종 |
| 상품 검색 바 | 위치 내 상품 이름/바코드 검색 |
| 상품 목록 | 상품 이미지 / 옵션(색상·사이즈) / 바코드 / 브랜드명 / 전산 수량 |

**Step 3: 재고조정 입력**

상품 선택 후 표시:

| 항목 | 설명 |
|------|------|
| 전산 수량 | 현재 전산재고 (변경 불가, 참조용) |
| 실제 수량 | 실물 확인 후 직접 입력 (숫자 인풋) |
| 조정 수량 | 자동 계산: 실제 수량 - 전산 수량 (양수: 파란색 +N, 음수: 빨간색 -N) |
| 조정 사유 | 텍스트 인풋 (예: 파손 제품 발견, 재고 실사 결과 추가 발견) |

**버튼**

| 버튼 | 설명 |
|------|------|
| 돌아가기 | 상품 목록으로 복귀 |
| 재고 조정 요청 | 실제 수량 입력 시 활성화. 클릭 시 조정 요청 전송 |

---

### 7-2. 작업자 화면 — 조정내역

**필터**

| 필드명 | 설명 |
|--------|------|
| 조회 기간 | 기간 범위 (기본: 최근 7일) |
| 승인 여부 | 전체 / 대기중 / 승인됨 / 거절됨 |

**조정 요청 내역 목록**

| 항목 | 설명 |
|------|------|
| 승인 여부 배지 | 승인됨(Green) / 대기중(Orange) / 거절됨(Red) |
| 요청일시 | 조정 요청 일시 |
| 상품 이미지 + 상품명 + 옵션 + 바코드 + 브랜드명 | SKU 정보 |
| 존 태그 + 위치코드 | 조정 위치 |
| 전산 수량 | 조정 시점 전산 수량 |
| 실제 수량 | 작업자가 입력한 실제 수량 |
| 조정 수량 | 전산 - 실제 (빨간색 음수 / 파란색 양수) |
| 사유 | 입력된 조정 사유 |

**상세 보기 — 승인된 항목**
- 상단 안내: "이 항목은 이미 승인되어 수정할 수 없습니다." (초록 배너)
- 전산/실제/조정 수량 표시 (수정 불가)
- 버튼: [돌아가기] 만 표시

**상세 보기 — 미승인/거절 항목 (재요청 가능)**
- 실제 수량 인풋 (재입력 가능)
- 조정 수량 자동 재계산
- 사유 인풋
- 버튼: [돌아가기] [재고 조정 재요청]

> 관리자 로그인 시 탭5에서 작업자 화면 대신 관리자 화면 표시

**요청 목록 필터**
