# 페이지 분석 — 바코드가입고

## 1. 페이지 개요

| 항목 | 내용 |
|------|------|
| 메뉴 그룹/위치 | 재고 조사 > 상품 가입고 |
| 페이지명 | 바코드가입고 |
| 식별 코드 | 미기재 |
| 목적 | 핸디터미널(HT) 또는 USB 바코드 스캐너를 이용하여 상품 바코드를 연속 스캔하는 방식으로 가입고 수량을 빠르게 등록하는 메뉴. 패션 창고의 대량 재고조사에 가장 많이 사용되는 입력 방식 |
| 접근 권한 | 물류사 작업자, 물류사 관리자, 시스템 관리자 |

> **패션 WMS 특이사항**: 의류는 동일 상품도 색상·사이즈별로 다른 바코드를 사용(컬러·사이즈 바코드 체계). 스캔 시 즉시 SKU를 특정할 수 있어 재고조사의 정확도와 속도를 모두 확보할 수 있다. 행거 타입 의류는 핸디터미널 스캔이 가장 효율적이다.

---

## 2. 업무 흐름 (Business Flow)

```
[재고조사 진행 중 — 창고 현장]
    ↓
[바코드가입고] 진입 (PC 또는 핸디터미널)
    ↓
화주 · 창고 · 조사차수 · 로케이션 설정
    ↓
스캐너를 바코드에 대고 스캔
    ↓
스캔 즉시 상품 정보 표시 + 스캔 누적 목록 업데이트
(동일 바코드 재스캔 시 수량 자동 +1)
    ↓
로케이션 이동 시 로케이션 변경 후 계속 스캔
    ↓
[저장] → 서버에 가입고 등록 (R-STK-603 반영)
```

## 3. 화면 구성

### 3-1. 헤더 설정 영역

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 화주 | Select | ✅ | 가입고 대상 화주 |
| 창고 | Select | ✅ | 가입고 대상 창고 |
| 조사 차수 | Select | ✅ | 현재 진행 중인 재고조사 회차 |
| 로케이션 | Input + Select | - | 현재 작업 중인 로케이션. 변경 시 이후 스캔부터 해당 로케이션으로 적용 |
| 스캔 모드 | Radio | - | 수량 자동 누산 / 1회 스캔 = 1 단위 |

---

### 3-2. 스캔 입력 영역

| 요소 | 설명 |
|------|------|
| 바코드 입력 필드 | 항상 포커스 유지. 스캐너 입력 또는 직접 타이핑 가능 |
| 스캔 피드백 | 스캔 성공 시: 초록색 플래시 + 성공음 / 실패(미등록) 시: 빨간색 플래시 + 경고음 |
| 마지막 스캔 결과 | 가장 최근 스캔된 상품 정보 크게 표시: 상품명, 스타일코드, 색상, 사이즈, 바코드, 현재 누산 수량 |

---

### 3-3. 스캔 누적 목록

| 컬럼명 | 설명 | 수정 |
|--------|------|------|
| 스캔순번 | 스캔된 순서 | ✗ |
| 바코드 | EAN-13 | ✗ |
| 브랜드 | 브랜드명 | ✗ |
| 스타일코드 | 스타일 코드 | ✗ |
| 색상 | 색상명 | ✗ |
| 사이즈 | 사이즈 표기 | ✗ |
| 상품명 | 상품 풀네임 | ✗ |
| 로케이션 | 스캔 시점의 로케이션 | ✅ |
| 스캔 횟수 | 해당 바코드가 스캔된 누적 횟수 (= 가입고 수량) | ✅ |
| 마지막 스캔 시각 | 가장 최근 스캔 시각 | ✗ |
| 삭제 | 해당 행 삭제 버튼 | - |

> 동일 바코드 재스캔 시 스캔 횟수 +1 누산. 실수 스캔 시 수량 직접 수정 또는 행 삭제 가능.

---

### 3-4. 실시간 요약

| 항목 | 설명 |
|------|------|
| 총 스캔 건수 (SKU 종류) | 누적 목록의 고유 SKU 수 |
| 총 스캔 수량 | 누산된 가입고 수량 합계 |
| 마지막 저장 시각 | 자동 임시저장 또는 마지막 수동 저장 시각 |

---

### 3-5. 액션 버튼

| 버튼명 | 권한 | 동작 |
|--------|------|------|
| 저장 | 작업자 이상 | 누적 목록 전체를 가입고로 서버 저장 |
| 임시저장 | 작업자 이상 | 서버에 임시저장 (이후 재개 가능) |
| 전체 삭제 | 작업자 이상 | 누적 목록 초기화 (확인 팝업) |
| 가입고 목록으로 | 전체 | R-STK-603으로 이동 |

---

## 4. 스캔 모드 설명

| 모드 | 동작 방식 | 적합 상황 |
|------|----------|----------|
| 자동 누산 모드 | 동일 바코드 연속 스캔 시 수량 자동 +1 | 동일 SKU가 여러 개 연속으로 있을 때 빠른 카운팅 |
| 단위 모드 | 모든 스캔을 개별 행으로 추가 | 스캔 이력을 세세히 추적해야 할 때 |

---

## 5. 입력 검증 규칙

| 상황 | 처리 방식 |
|------|----------|
| 미등록 바코드 스캔 | 경고음 + "등록되지 않은 바코드입니다: {barcode}" 토스트 |
| 수량 0 또는 음수 수동 입력 | "수량은 1 이상이어야 합니다." 검증 에러 |
| 로케이션 미입력 상태에서 저장 | 경고 팝업: "로케이션이 입력되지 않은 항목이 있습니다. 계속 저장하시겠습니까?" |
| 저장 전 페이지 이탈 | "저장되지 않은 스캔 데이터가 있습니다. 이탈 시 데이터가 사라집니다." 경고 |

---

## 6. 핸디터미널(HT) 지원 고려사항

- 화면 레이아웃은 태블릿/모바일 웹에서도 사용 가능하도록 반응형 구현
- 스캔 입력 필드가 항상 최상단에 노출되어 한 손 스캔 가능
- 자동 임시저장: 100건 스캔마다 자동으로 임시저장 (네트워크 단절 대비)

## 8. API/이벤트

| API | Method | 설명 |
|-----|--------|------|
| `/api/products/barcode/{barcode}` | GET | 바코드로 상품 정보 즉시 조회 (스캔 즉시 호출) |
| `/api/stock/pre-receipt/barcode-scan/temp-save` | POST | 임시저장 |
| `/api/stock/pre-receipt/barcode-scan/confirm` | POST | 최종 저장 (가입고 등록) |
