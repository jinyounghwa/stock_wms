## 1. 메뉴 분리: 재고 조정 / 재고 조사

현재 두 기능이 하나의 메뉴로 합쳐져 있으므로 **반드시 분리** 필요.

### 재고 조정 (앱 — PDA)

- **용도**: 단건 재고 불일치 확인 및 수정 요청
- **시나리오**: 출고 지시 후 일정 기간(예: 1주일) 출고가 안 나가는 경우, 해당 위치로 가서 특정 상품만 실물 확인
- **프로세스**
  1. 작업자가 해당 로케이션으로 이동
  2. 대상 상품 바코드 스캔 또는 수량 직접 입력
  3. 재고 조정 요청을 관리자에게 전송
  4. 관리자가 요청 내용 검토 후 승인/반려

### 재고 조사 

- **용도**: 대규모 일괄 재고 실사
- **대상 예시**: 전 센터 통합 재고 조사, 특정 브랜드 전체 재고 조사, 시즌 상품(여름 등) 전체 재고 조사
- **프로세스**
  1. 수십 명의 작업자에게 PDA 배포
  2. 지정 영역의 모든 상품을 순서대로 전수 스캔
  3. 전산 재고 리스트 vs 실사 스캔 리스트 차이 비교 (과부족 확인)
  4. 하루~이틀 일정이 모두 종료된 후, 관리자가 최종 검토 및 재고 반영

## 2. 위치별 재고 화면 개선

### 뷰 전환 추가

- 기존 **목록(리스트) 뷰**에 더해 **대시보드 뷰**를 추가
- 목록 클릭 시 상세 정보 표시하는 기존 구조 유지

### 로케이션 바코드 스캔 기능

- 로케이션 코드 입력란에서 **바코드 스캔**으로 즉시 해당 위치의 전산 재고와 실재고를 비교 확인할 수 있도록 지원
- PDA에서 로케이션 바코드를 찍으면 바로 해당 위치 재고 조회

---

## 3. 부진 재고 화면

- 대시보드 성격의 화면이 적합 (기존 방향 유지)
- 구성 요소: 위치별 재고 비중, 창고/존별 현황 등

---

## 4. 존(Zone)별 재고 시각화 (검토 필요)

회의에서 여러 시각화 방안이 제시되었으나, 최종 확정 전 추가 논의가 필요한 항목.

### 제안된 방안

- **히트맵/버블맵**: 존별 재고량을 원 크기로 표현, 물리적 위치 기반 배치
- **네비게이션 맵**: 창고 도면(2D 지도) 위에 존 위치와 재고량 표시
- **단(段)별 적재율**: 하나의 컬럼(랙) 기준 1단~5단 각 단의 적재 현황 시각화
- 차트 라이브러리 활용 검토

### 논의 사항

- 현재 창고 도면이 2D이므로, 1층~5층(1단~5단) 및 각 칸의 적재율 표현 방법 고민 필요
- 물리적 창고 지도 이미지 위에 직접 표시하는 방식도 가능
- **우선순위는 기본 기능 완성 후** 추가 개발 검토

---

## 5. 입출고 목록 (로그 페이지 - 이건 확인 후 데모 개발)

- 스프린트 계획상 **마지막 순서**로 배치
- 모든 입고, 출고, 이동 데이터를 **단순 리스트 형태**로 표시하는 로그 페이지
- 현재 파트에서 즉시 작업하지 않아도 무방

---

## 6. 전체 진행 방향 합의

| 항목 | 내용 |
|------|------|
| 작업 방식 | 데모 먼저 제작 → 피드백 반영하여 기능 추가/삭제 → 확정 후 기획서 작성 |
| 우선순위 | 기본 기능 먼저 완성, 시각화 등 부가 기능은 후순위 |
| 다음 미팅 | 수정된 데모를 기반으로 재검토 예정 |

---

## 7. 액션 아이템 요약

1. **재고 조정 / 재고 조사 메뉴 분리** — 즉시 반영
2. **위치별 재고 화면에 대시보드 뷰 추가** 및 로케이션 스캔 기능 적용
3. **부진 재고 대시보드** 다듬기
4. **존별 시각화** — 추가 논의 후 확정 (우선순위 낮음)
5. **입출고 로그 페이지** — 스프린트 마지막 단계에서 진행
6. **수정된 데모** 재공유 후 다음 미팅에서 확정
