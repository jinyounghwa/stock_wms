# 재고관리 데모페이지 수정개선내역

## 1. 이번 정리 방향

기존 문서가 회의 메모 성격이었다면, 이번 문서는 현재 구축된 프로젝트 화면을 기준으로 **업무 계산식 중심**으로 다시 정리한 요약본이다.

즉, 이번 문서에서는 아래처럼 설명한다.

- 무엇을 더한 값인가
- 무엇을 뺀 값인가
- 무엇을 나눈 뒤 100을 곱한 값인가
- 어떤 값을 가중치로 곱해서 더한 값인가

---

## 2. 현재 화면 기준 주요 개선 방향

### 2-1. 재고조사와 재고조정 분리

현재 프로젝트는 재고조사와 재고조정을 분리해서 본다.

#### 재고조사

- 실제 조사 수량을 모은다.
- 현재고와 비교한다.
- 차이 수량을 계산한다.

핵심 계산식:

- `차이 수량 = 조사 수량 - 현재고`

#### 재고조정

- 실제 수량과 전산 수량의 차이를 조정 요청으로 만든다.
- 승인/반려로 처리한다.

핵심 계산식:

- `조정 수량 = 실제 수량 - 전산 수량`

즉, 현재 구조는

- 재고조사 = 실사 비교
- 재고조정 = 차이 반영 요청

으로 분리된 상태다.

### 2-2. 위치별 재고현황의 시각화 강화

현재 위치별 재고현황은 단순 목록만 보는 것이 아니라,

- 총재고
- 가용
- 예약
- 적재 비율

을 함께 본다.

핵심 계산식:

- `적재 비율 = (현재 적재 수량 / 적재 가능 수량) × 100`
- `예약 비율 = (예약 수량 / 총재고) × 100`

즉, 단순히 “몇 개 있다”가 아니라 “얼마나 차 있는가”와 “얼마나 묶여 있는가”까지 보도록 개선된 상태다.

### 2-3. 부진재고를 분석형 화면으로 정리

현재 부진재고는 오래된 재고를 목록으로만 보여주는 방식이 아니다.

아래를 함께 본다.

- 무출고 기간
- 입고 후 경과 기간
- 수량 규모
- 위치 중요도

핵심 계산식:

- `위치 수량 = 가용 수량 + 예약 수량`
- `위치 비중 = (위치 수량 / SKU 총재고) × 100`
- `우선순위 점수 = (경과일 가중치 × 0.42 + 존 가중치 × 0.28 + 수량 가중치 × 0.20 + 가용 비중 × 0.10) × 100`

즉, 현재 부진재고는 단순 조회가 아니라 **우선 처리 대상을 고르는 화면**으로 개선된 상태다.

## 3. 현재 메뉴별 핵심 계산값

### 3-1. 재고조사

#### 조사 수량

- 같은 상품, 같은 위치에 대해 입력된 수량을 모두 더한 값
- `조사 수량 = 입력1 + 입력2 + 입력3 + ...`

#### 현재고

- 시스템에 등록된 위치별 재고 수량

#### 차이 수량

- `차이 수량 = 조사 수량 - 현재고`

#### 재고 정확도

- `재고 정확도 = (일치 건수 / 전체 비교 건수) × 100`

### 3-2. 재고조정

#### 전산 수량

- 시스템이 가지고 있는 현재 수량

#### 실제 수량

- 현장에서 다시 센 수량

#### 조정 수량

- `조정 수량 = 실제 수량 - 전산 수량`

### 3-3. 위치별 재고현황

#### 총재고 합계

- `총재고 합계 = 위치1 총재고 + 위치2 총재고 + ...`

#### 총 가용 수량

- `총 가용 수량 = 위치1 가용 + 위치2 가용 + ...`

#### 총 예약 수량

- `총 예약 수량 = 위치1 예약 + 위치2 예약 + ...`

#### 평균 적재 비율

- `평균 적재 비율 = (위치1 적재 비율 + 위치2 적재 비율 + ... ) / 위치 수`

#### 예약 비율

- `예약 비율 = (총 예약 수량 / 총재고 합계) × 100`

### 3-4. 부진재고

#### SKU 총재고

- `SKU 총재고 = 위치1 수량 + 위치2 수량 + ...`

#### 위치 수량

- `위치 수량 = 가용 수량 + 예약 수량`

#### 위치 비중

- `위치 비중 = (위치 수량 / SKU 총재고) × 100`

#### 우선순위 점수

- `우선순위 점수 = (경과일 가중치 × 0.42 + 존 가중치 × 0.28 + 수량 가중치 × 0.20 + 가용 비중 × 0.10) × 100`

## 4. 현재 프로젝트 기준으로 보면 좋은 점

현재 프로젝트는 계산 구조가 비교적 명확하다.

### 4-1. 재고조사

- 실사 수량과 현재고의 차이를 바로 볼 수 있다.

### 4-2. 재고조정

- 조정 수량이 “실제 수량 - 전산 수량”으로 명확하다.

### 4-3. 위치별 재고현황

- 수량뿐 아니라 적재 비율과 예약 비율까지 같이 볼 수 있다.

### 4-4. 부진재고

- 오래된 재고를 단순 검색이 아니라 점수화해서 우선순위를 정할 수 있다.

## 5. 앞으로 입출고 목록은 어떻게 만들면 좋은가

입출고 목록은 단순한 로그가 아니라 재고 변동 원장으로 가는 것이 좋다.

핵심 계산식은 아래 3개면 충분하다.

### 이전 수량

- 변경 전 수량

### 변동 수량

- 증가 또는 감소된 수량

### 변경 후 수량

- `변경 후 수량 = 이전 수량 + 변동 수량`

추가로 보고 싶은 값은 아래다.

#### 총 입고 수량

- `총 입고 수량 = 증가한 수량 합계`

#### 총 출고 수량

- `총 출고 수량 = 감소한 수량 절대값 합계`

#### 순변동 수량

- `순변동 수량 = 총 입고 수량 - 총 출고 수량`

즉, 입출고 목록은 결국 모든 기능의 결과를 한 곳에서 설명하는 구조가 되어야 한다.

## 6. 최종 요약

현재 프로젝트를 계산식으로만 다시 요약하면 아래와 같다.

- 재고조사 = `조사 수량 - 현재고`
- 재고정확도 = `(일치 건수 / 전체 건수) × 100`
- 재고조정 = `실제 수량 - 전산 수량`
- 위치별 적재 비율 = `(현재 적재 수량 / 적재 가능 수량) × 100`
- 위치별 예약 비율 = `(예약 수량 / 총재고) × 100`
- 부진재고 위치 비중 = `(위치 수량 / SKU 총재고) × 100`
- 부진재고 우선순위 = 여러 기준값에 가중치를 곱해 더한 점수
- 입출고 변경 후 수량 = `이전 수량 + 변동 수량`
