# 현재 구축 프로젝트 기준 재고관리 설명문서

## 1. 문서 목적

이 문서는 현재 구축된 프로젝트 화면을 기준으로 재고관리 기능을 설명한 문서이다.

이번 문서의 핵심 원칙은 아래와 같다.

- 개발 함수명이나 컴포넌트명이 아니라 **업무에서 이해할 수 있는 계산식**으로 설명한다.
- 각 값이 **무엇을 더한 값인지, 무엇을 뺀 값인지, 어떤 비율인지**를 명확히 적는다.
- 화면에 보이는 수치가 어떤 기준으로 나오는지 **업무 용어로 해석**한다.

즉, “어느 코드에서 계산한다”가 아니라 “어떤 값에 어떤 값을 더하고 빼서 이 숫자가 나오는가”를 기준으로 정리한다.

---

## 2. 공통 계산 원칙

### 2-1. 재고 계산의 기본 단위

현재 프로젝트에서 재고는 아래 두 축으로 본다.

- 상품
- 위치

즉, 같은 상품이라도 위치가 다르면 별도 재고로 본다.

예를 들어 같은 바코드 상품이

- A-01-01 위치에 10개
- B-02-03 위치에 20개

있으면,

- 상품 총재고는 `10 + 20 = 30`
- 위치별 재고는 각각 따로 본다.

### 2-2. 합계, 차이, 비율의 기본 원리

문서에 나오는 대부분의 계산은 아래 3가지다.

#### 1) 합계

- `합계 = 항목1 + 항목2 + 항목3 + ...`

#### 2) 차이

- `차이 = 실제값 - 기준값`

#### 3) 비율

- `비율(%) = (부분값 / 전체값) × 100`

즉, 이 문서의 모든 KPI는 결국 **더하기**, **빼기**, **나누기 후 100 곱하기**로 설명된다.

## 3. 재고조사

재고조사는 실제로 세어 본 수량과 시스템에 잡혀 있는 수량을 비교하는 기능이다.

재고조사의 핵심 질문은 아래다.

- 실제로 세어 본 수량은 몇 개인가?
- 시스템에는 몇 개로 잡혀 있는가?
- 둘의 차이는 몇 개인가?

### 3-1. 전체 조사 건수

전체 조사 건수는 저장된 조사 데이터의 개수다.

- `전체 조사 건수 = 조사 데이터 1건 수 + 조사 데이터 2건 수 + ...` 가 아니라
- `전체 조사 건수 = 조사 행의 개수`

즉, 수량 합이 아니라 **입력된 조사 기록의 개수**다.

예시:

- 조사 데이터가 15줄 있으면 전체 조사 건수는 `15건`

### 3-2. 조사 진행중 건수

조사 진행중 건수는 상태가 “조사중”인 행의 개수다.

- `조사 진행중 건수 = 조사중 행 개수`

예시:

- 전체 15건 중 9건이 조사중이면 `9건`

### 3-3. 실입고 완료 건수

실입고 완료 건수는 상태가 “실입고완료”인 행의 개수다.

- `실입고 완료 건수 = 실입고완료 행 개수`

예시:

- 전체 15건 중 6건이 실입고완료면 `6건`

### 3-4. 조사 수량

조사 수량은 같은 상품, 같은 위치에 대해 조사자가 입력한 수량을 모두 더한 값이다.

- `조사 수량 = 1차 입력 수량 + 2차 입력 수량 + 추가 입력 수량`

예시:

- 같은 상품/위치에 10개 입력
- 다시 5개 입력
- 다시 3개 입력

이면

- `조사 수량 = 10 + 5 + 3 = 18`

즉, 현재 프로젝트에서 조사 수량은 마지막 1건이 아니라 **누적 합계**다.

### 3-5. 현재고

현재고는 시스템에 저장된 위치별 재고 수량이다.

쉽게 말하면,

- “전산상 이 위치에 몇 개 있다고 보고 있느냐”를 뜻한다.

예시:

- 시스템이 A-01-01 위치에 20개 있다고 가지고 있으면
- 현재고는 `20`

### 3-6. 차이 수량

재고조사에서 가장 중요한 값은 차이 수량이다.

- `차이 수량 = 조사 수량 - 현재고`

예시 1:

- 조사 수량 `18`
- 현재고 `20`

이면

- `차이 수량 = 18 - 20 = -2`

예시 2:

- 조사 수량 `25`
- 현재고 `20`

이면

- `차이 수량 = 25 - 20 = +5`

해석은 아래와 같다.

- `0`이면 일치
- `양수`이면 실제가 더 많음
- `음수`이면 실제가 더 적음

### 3-7. 대조 결과

대조 결과는 차이 수량을 상태로 바꾼 것이다.

- `차이 수량 = 0` → 일치
- `차이 수량 > 0` → 불일치(+)
- `차이 수량 < 0` → 불일치(-)
- 조사 자체가 없으면 → 미처리

즉, 대조 결과는 새로운 계산값이 아니라 **차이 수량의 해석 결과**다.

### 3-8. 일치 건수

일치 건수는 차이 수량이 0인 행의 개수다.

- `일치 건수 = 차이 수량이 0인 행 개수`

### 3-9. 불일치(+) 건수

불일치(+) 건수는 조사 수량이 현재고보다 큰 행의 개수다.

- `불일치(+) 건수 = (조사 수량 - 현재고) > 0 인 행 개수`

### 3-10. 불일치(-) 건수

불일치(-) 건수는 조사 수량이 현재고보다 작은 행의 개수다.

- `불일치(-) 건수 = (조사 수량 - 현재고) < 0 인 행 개수`

### 3-11. 미처리 건수

미처리 건수는 조사 수량이 아직 입력되지 않은 비교 대상의 개수다.

- `미처리 건수 = 조사 기록이 없는 행 개수`

즉, 미처리는 수량 비교 이전 단계다.

### 3-12. 재고 정확도

재고 정확도는 비교 대상 중 얼마나 많은 건이 정확히 일치했는지를 퍼센트로 본 값이다.

- `재고 정확도(%) = (일치 건수 / 전체 비교 건수) × 100`

예시:

- 전체 비교 건수 `100`
- 일치 건수 `92`

이면

- `재고 정확도 = (92 / 100) × 100 = 92%`

### 3-13. 최소 차이 필터

최소 차이 필터는 차이 수량의 절대값이 일정 기준 이상인 것만 보는 방식이다.

- `필터 통과 조건 = |차이 수량| >= 기준값`

예시:

- 기준값 `5`
- 차이 수량 `+3` → 제외
- 차이 수량 `-4` → 제외
- 차이 수량 `+5` → 포함
- 차이 수량 `-7` → 포함

즉, 플러스/마이너스 방향보다 **차이의 크기**를 보는 필터다.

### 3-14. 재고조정 연계 수량

재고조사에서 재고조정으로 넘길 때 조정 기준 수량은 아래와 같다.

- `조정 기준 수량 = 조사 수량 - 현재고`

즉, 재고조사에서 계산한 차이 수량이 그대로 재고조정의 출발값이 된다.

## 4. 재고조정

재고조정은 실제 수량과 시스템 수량의 차이를 반영 요청하는 기능이다.

재고조정의 핵심 질문은 아래다.

- 시스템은 몇 개라고 보고 있는가?
- 실제로는 몇 개인가?
- 그래서 몇 개를 늘리거나 줄여야 하는가?

### 4-1. 전체 요청 건수

전체 요청 건수는 등록된 조정 요청 행의 개수다.

- `전체 요청 건수 = 조정 요청 행 개수`

### 4-2. 대기중 건수

- `대기중 건수 = 상태가 대기중인 요청 행 개수`

### 4-3. 승인됨 건수

- `승인됨 건수 = 상태가 승인됨인 요청 행 개수`

### 4-4. 거절됨 건수

- `거절됨 건수 = 상태가 거절됨인 요청 행 개수`

### 4-5. 전산 수량

전산 수량은 현재 시스템이 해당 상품/위치에 있다고 보고 있는 수량이다.

즉,

- “지금 전산상 몇 개냐”를 뜻한다.

### 4-6. 실제 수량

실제 수량은 작업자가 현장에서 직접 세어 입력한 수량이다.

즉,

- “실제로 지금 몇 개냐”를 뜻한다.

### 4-7. 조정 수량

재고조정의 핵심 값은 조정 수량이다.

- `조정 수량 = 실제 수량 - 전산 수량`

예시 1:

- 실제 수량 `12`
- 전산 수량 `10`

이면

- `조정 수량 = 12 - 10 = +2`

예시 2:

- 실제 수량 `7`
- 전산 수량 `10`

이면

- `조정 수량 = 7 - 10 = -3`

해석:

- `양수` → 재고를 늘려야 함
- `음수` → 재고를 줄여야 함
- `0` → 조정할 필요 없음

### 4-8. 조정 요청 가능 조건

현재 프로젝트에서 조정 요청은 아래일 때만 의미가 있다.

- 실제 수량이 입력되어 있어야 함
- 실제 수량과 전산 수량이 달라야 함

즉, 수식으로 쓰면 아래와 같다.

- `조정 요청 가능 = 실제 수량이 존재하고, (실제 수량 - 전산 수량) ≠ 0`

### 4-9. 재요청 수량

재요청도 계산 원리는 동일하다.

- `재요청 조정 수량 = 다시 입력한 실제 수량 - 기존 전산 수량`

즉, 재요청은 기존 요청을 다시 계산하는 것이 아니라 **새로 입력한 실제 수량으로 차이를 다시 구하는 것**이다.

### 4-10. 승인/반려의 의미

현재 프로젝트에서 승인/반려는 수량을 다시 계산하는 단계라기보다 상태를 확정하는 단계다.

- 승인 → 이 차이를 인정함
- 반려 → 이 차이를 인정하지 않음

즉, 현재 구현에서는 승인/반려 자체보다 그 전에 계산한

- `실제 수량 - 전산 수량`

이 핵심 값이다.

## 5. 위치별 재고현황

위치별 재고현황은 위치마다 얼마나 재고가 쌓여 있는지, 그 중 얼마나 바로 쓸 수 있고 얼마나 묶여 있는지를 보여주는 화면이다.

핵심 질문은 아래다.

- 이 위치에 총 몇 개가 있는가?
- 그 중 바로 쓸 수 있는 것은 몇 개인가?
- 이미 예약된 것은 몇 개인가?
- 이 위치는 얼마나 차 있는가?

### 5-1. 총재고

총재고는 그 위치에 쌓여 있는 전체 수량이다.

실무적으로는 아래처럼 이해하면 된다.

- `총재고 = 가용 + 예약 + 기타 상태 재고`

현재 화면에서는 주로 총재고값 자체를 기준값으로 사용한다.

### 5-2. 가용재고

가용재고는 바로 출고나 운영에 사용할 수 있는 수량이다.

즉,

- “지금 당장 정상적으로 쓸 수 있는 재고”

다.

### 5-3. 예약재고

예약재고는 이미 주문이나 작업에 배정되어 자유롭게 쓸 수 없는 수량이다.

즉,

- “있기는 하지만 이미 다른 작업에 묶여 있는 재고”

다.

### 5-4. 위치 수

위치 수는 현재 조회 결과에 포함된 위치의 개수다.

- `위치 수 = 조회된 위치 행 개수`

### 5-5. 총 재고 합계

여러 위치를 묶어서 볼 때 총 재고 합계는 아래와 같다.

- `총 재고 합계 = 위치1 총재고 + 위치2 총재고 + 위치3 총재고 + ...`

### 5-6. 총 가용 수량

- `총 가용 수량 = 위치1 가용 + 위치2 가용 + 위치3 가용 + ...`

### 5-7. 총 예약 수량

- `총 예약 수량 = 위치1 예약 + 위치2 예약 + 위치3 예약 + ...`

### 5-8. 적재 비율

적재 비율은 해당 위치가 수용 가능량 대비 얼마나 차 있는지를 백분율로 나타낸 값이다.

- `적재 비율(%) = (현재 적재 수량 / 위치 적재 가능 수량) × 100`

예시:

- 현재 적재 수량 `80`
- 위치 적재 가능 수량 `100`

이면

- `적재 비율 = (80 / 100) × 100 = 80%`

### 5-9. 평균 적재 비율

여러 위치를 한 번에 볼 때 평균 적재 비율은 각 위치 적재 비율의 평균이다.

- `평균 적재 비율 = (위치1 적재 비율 + 위치2 적재 비율 + ... ) / 위치 수`

예시:

- 위치1 적재 비율 `50%`
- 위치2 적재 비율 `70%`
- 위치3 적재 비율 `80%`

이면

- `평균 적재 비율 = (50 + 70 + 80) / 3 = 66.7%`

### 5-10. 예약 비율

예약 비율은 총재고 중 예약재고가 차지하는 비중이다.

- `예약 비율(%) = (총 예약 수량 / 총 재고 합계) × 100`

예시:

- 총 예약 수량 `30`
- 총 재고 합계 `200`

이면

- `예약 비율 = (30 / 200) × 100 = 15%`

### 5-11. 선택 위치 요약

현재 화면은 사용자가 특정 위치를 선택하면 선택한 위치들만 다시 합산해서 보여준다.

즉,

- 선택한 위치가 있으면 → 선택 위치만 더한 값
- 선택한 위치가 없으면 → 조회된 전체 위치를 더한 값

으로 계산한다.

### 5-12. 적재 강도 구간

현재 화면은 적재 비율을 구간으로 나누어 시각 상태를 준다.

- `0% 이하` → 비어있음
- `0% 초과 ~ 30% 이하` → 낮음
- `30% 초과 ~ 60% 이하` → 보통
- `60% 초과 ~ 90% 이하` → 높음
- `90% 초과` → 가득 참

즉, 카드 색이나 강도는 감이 아니라 **적재 비율 구간**으로 결정된다.

## 6. 부진재고

부진재고는 오랫동안 움직이지 않은 재고를 찾고, 그 재고가 얼마나 비효율적으로 공간을 차지하는지를 판단하는 기능이다.

핵심 질문은 아래다.

- 얼마나 오랫동안 출고가 없었는가?
- 현재 몇 개가 쌓여 있는가?
- 어느 위치를 얼마나 점유하고 있는가?
- 우선적으로 처리해야 할 재고인가?

### 6-1. SKU 총재고

부진재고는 먼저 상품 단위 총재고를 계산한다.

- `SKU 총재고 = 위치1 수량 + 위치2 수량 + 위치3 수량 + ...`

여기서 각 위치 수량은 아래다.

- `위치 수량 = 가용 수량 + 예약 수량`

즉,

- `SKU 총재고 = (위치1 가용 + 위치1 예약) + (위치2 가용 + 위치2 예약) + ...`

### 6-2. 부진재고 필터 조건

현재 프로젝트에서 부진재고 대상은 아래를 함께 만족해야 한다.

- 무출고일수가 기준 이상
- 입고 후 경과일수가 기준 이상
- 총재고가 기준 이상

즉, 계산식으로 쓰면 아래와 같다.

- `무출고일수 >= 기준 무출고일수`
- `입고후 경과일수 >= 기준 입고후 경과일수`
- `SKU 총재고 >= 최소 재고 기준`

### 6-3. 위치 비중

한 상품이 여러 위치에 나뉘어 있을 때, 특정 위치가 전체 수량 중 얼마나 차지하는지 보는 값이다.

- `위치 비중(%) = (해당 위치 수량 / SKU 총재고) × 100`

예시:

- SKU 총재고 `100`
- 특정 위치 수량 `25`

이면

- `위치 비중 = (25 / 100) × 100 = 25%`

### 6-4. 부진 등급

현재 부진 등급은 무출고일수 기준으로 나눈다.

- `365일 이상` → 위험
- `180일 이상 365일 미만` → 경고
- `90일 이상 180일 미만` → 주의
- `90일 미만` → 정상

즉, 무출고일수가 길수록 등급이 올라간다.

### 6-5. 부진재고 수량 요약

부진재고 요약은 아래처럼 계산한다.

#### 부진 SKU 수

- `부진 SKU 수 = 조건을 만족한 상품 개수`

#### 부진 가용 수량 합계

- `부진 가용 수량 합계 = 위치1 가용 + 위치2 가용 + ...`

#### 부진 예약 수량 합계

- `부진 예약 수량 합계 = 위치1 예약 + 위치2 예약 + ...`

### 6-6. 우선순위 점수

현재 부진재고에서 가장 중요한 값은 우선순위 점수다.

이 점수는 아래 4가지를 합쳐 만든다.

- 얼마나 오래 안 나갔는가
- 얼마나 중요한 존을 점유하고 있는가
- 수량이 얼마나 큰가
- 가용 수량 비중이 얼마나 큰가

각 요소는 아래처럼 계산한다.

#### 1) 경과일 가중치

- `경과일 가중치 = 무출고일수 / 365`
- 단, 최대값은 `1`

즉, 365일 이상이면 이 항목은 최대치로 본다.

#### 2) 존 가중치

존 중요도에 따라 미리 정한 값을 쓴다.

- 출고존 = `1`
- 입고존 = `0.88`
- 보관존 = `0.46`
- 반품존 = `0.34`
- 뮬랑 = `0.26`

즉, 중요한 존을 막고 있을수록 점수가 높아진다.

#### 3) 수량 가중치

- `수량 가중치 = 해당 위치 수량 / 비교 대상 중 최대 위치 수량`

즉, 같은 화면 안에서 가장 많은 위치를 1로 보고 상대 비율을 계산한다.

#### 4) 가용 비중

- `가용 비중 = 가용 수량 / 위치 수량`

즉, 쌓여 있는 수량 중 실제로 처리 가능한 비중을 본다.

#### 5) 최종 우선순위 점수

- `우선순위 점수 = (경과일 가중치 × 0.42 + 존 가중치 × 0.28 + 수량 가중치 × 0.20 + 가용 비중 × 0.10) × 100`

마지막에 반올림해서 정수 점수로 본다.

즉, 이 값은 단순히 오래된 재고가 아니라

- 오래됐고
- 중요한 자리를 차지하고
- 수량도 많고
- 실제로 처리 가능한 물량이 많은 재고

일수록 높아진다.

### 6-7. 조치 수준

우선순위 점수를 아래처럼 해석한다.

- `75점 이상` → 즉시 이동
- `60점 이상 75점 미만` → 우선 이동
- `45점 이상 60점 미만` → 모니터링
- `45점 미만` → 관찰

### 6-8. 고위험 위치 수

- `고위험 위치 수 = 우선순위 점수가 높은 위치 개수`

현재 프로젝트에서는 보통 아래 기준으로 본다.

- `priorityScore >= 70`

### 6-9. 우선 이동 위치 수

- `우선 이동 위치 수 = priorityScore >= 60 인 위치 개수`

### 6-10. 핵심 존 적치 수량

핵심 존 적치 수량은 출고존과 입고존에 쌓여 있는 부진재고 수량 합계다.

- `핵심 존 적치 수량 = 출고존 수량 합 + 입고존 수량 합`

즉, 창고 운영상 중요한 자리를 막고 있는 부진재고 규모를 보는 값이다.

### 6-11. 고위험 수량

- `고위험 수량 = 고위험 위치 수량 합계`

즉, 위험한 위치에 쌓인 실제 물량 규모다.

### 6-12. 평균 리스크 점수

- `평균 리스크 점수 = 전체 우선순위 점수 합 / 위치 개수`

예시:

- 위치1 점수 `80`
- 위치2 점수 `60`
- 위치3 점수 `40`

이면

- `평균 리스크 = (80 + 60 + 40) / 3 = 60`

### 6-13. 존별 우선 이동 수량

존별 요약에서는 우선 이동 대상 수량과 일반 수량을 나누어 본다.

#### 우선 이동 수량

- `우선 이동 수량 = priorityScore >= 60 인 위치 수량 합계`

#### 일반 수량

- `일반 수량 = 존 총수량 - 우선 이동 수량`

즉, 각 존마다 “지금 먼저 처리해야 할 물량”과 “아직 급하지 않은 물량”을 분리해서 보는 방식이다.

## 7. 입출고 목록은 어떻게 구성하는 것이 좋은가

현재 프로젝트의 주력 메뉴는 재고조사, 재고조정, 위치별 재고현황, 부진재고다.

이 흐름을 기준으로 하면 입출고 목록은 단순 조회가 아니라 **재고 변동 원장**으로 가는 것이 가장 좋다.

### 7-1. 입출고 목록 1행의 의미

입출고 목록의 1행은 “어느 시점에 재고가 왜 변했는가”를 보여줘야 한다.

즉, 한 줄에 아래가 보여야 한다.

- 언제 변했는가
- 어떤 상품인가
- 어느 위치인가
- 왜 변했는가
- 몇 개가 바뀌었는가
- 바뀌기 전에는 몇 개였는가
- 바뀐 후에는 몇 개인가

### 7-2. 입출고 목록 핵심 계산값

#### 이전 수량

- 변경 직전 수량

#### 변동 수량

- `변동 수량 = 증가 또는 감소된 수량`

#### 변경 후 수량

- `변경 후 수량 = 이전 수량 + 변동 수량`

예시:

- 이전 수량 `20`
- 변동 수량 `-3`

이면

- `변경 후 수량 = 20 + (-3) = 17`

### 7-3. 입고 합계

- `총 입고 수량 = 입고로 증가한 수량 합계`

즉, 플러스 방향의 변동을 모두 더한 값이다.

### 7-4. 출고 합계

- `총 출고 수량 = 출고로 감소한 수량 합계`

즉, 마이너스 방향의 변동 절대값을 더한 값이다.

예시:

- `-3`, `-7`, `-10`

이면

- `총 출고 수량 = 3 + 7 + 10 = 20`

### 7-5. 순변동 수량

- `순변동 수량 = 총 입고 수량 - 총 출고 수량`

즉, 조회 기간 동안 전체 재고가 순증했는지 순감했는지를 보여준다.

### 7-6. 재고조사/재고조정과의 연결

입출고 목록은 아래 이벤트가 발생할 때 함께 남는 구조가 좋다.

#### 재고조사 반영

- `변동 수량 = 조사 수량 - 현재고`

#### 재고조정 승인

- `변동 수량 = 실제 수량 - 전산 수량`

#### 위치 이동

- 출발 위치: `이전 수량 - 이동 수량`
- 도착 위치: `이전 수량 + 이동 수량`

즉, 입출고 목록은 결국 모든 기능의 결과를 한 곳에서 설명하는 화면이 되어야 한다.

## 8. 결론

현재 프로젝트 기준으로 각 메뉴의 계산 원리를 한 줄로 정리하면 아래와 같다.

- 재고조사 = `조사 수량 - 현재고`
- 재고정확도 = `(일치 건수 / 전체 건수) × 100`
- 재고조정 = `실제 수량 - 전산 수량`
- 위치별 재고현황 적재율 = `(현재 적재 수량 / 적재 가능 수량) × 100`
- 위치별 예약 비율 = `(예약 수량 / 총재고) × 100`
- 부진재고 위치 비중 = `(위치 수량 / SKU 총재고) × 100`
- 부진재고 우선순위 = 여러 가중치를 곱해 더한 점수
- 입출고 변경 후 수량 = `이전 수량 + 변동 수량`

즉, 이 프로젝트는 화면이 많아 보여도 결국 아래 4개의 계산 원리로 이해할 수 있다.

- 합계
- 차이
- 비율
- 가중합
