# 부진재고 시각화 산식 설명

## 1. 우선이동점수란?

우선이동점수는 부진재고가 현재 위치를 얼마나 비효율적으로 점유하고 있는지를 수치화한 값이다.  
오래 출고되지 않았는지, 중요한 존을 점유하고 있는지, 수량이 많이 쌓였는지, 바로 이동 가능한 재고가 많은지를 함께 반영해서 계산한다.

현재 프로젝트의 산식은 아래와 같다.

```text
우선이동점수
= {
  (미출고일수 ÷ 365 × 0.42)
+ (존 가중치 × 0.28)
+ (해당 위치 총수량 ÷ 최대 위치수량 × 0.20)
+ (가용수량 ÷ 해당 위치 총수량 × 0.10)
} × 100
```

마지막 계산 결과는 반올림해서 점수로 표시한다.

```text
priorityScore
= round(((ageWeight × 0.42)
+ (zoneWeight × 0.28)
+ (qtyWeight × 0.20)
+ (availableShare × 0.10)) × 100)
```

## 2. 우선이동점수의 각 항목 뜻

### 2-1. 미출고일수 ÷ 365

이 값은 재고가 얼마나 오래 출고되지 않았는지를 1년 기준 비율로 바꾼 값이다.  
`365`는 1년 기준값이며, 미출고일수를 0부터 1 사이 값으로 정규화하기 위한 분모다.

예를 들어 미출고일수가 240일이면:

```text
240 ÷ 365 = 0.6575
```

### 2-2. × 0.42

`0.42`는 미출고일수 항목을 전체 우선이동점수에 42% 비중으로 반영하겠다는 뜻이다.  
즉, 오래 안 나간 재고일수록 우선적으로 이동해야 한다는 판단을 가장 크게 반영한 가중치다.

예를 들어 미출고일수 비율이 `0.6575`이면:

```text
0.6575 × 0.42 = 0.2762
```

### 2-3. 존 가중치

존 가중치는 계산으로 새로 만들어지는 값이 아니라, 존 중요도에 따라 미리 정해 둔 고정값이다.  
위치명에서 `A`, `B`, `C`, `D` 같은 세부 구역은 빼고, 기본 존 이름만 보고 가중치를 적용한다.

예를 들어:

```text
입고존-D → 기본 존은 입고존 → 존 가중치 0.88
출고존-A → 기본 존은 출고존 → 존 가중치 1.00
보관존-H → 기본 존은 보관존 → 존 가중치 0.46
```

현재 프로젝트 기준 존 가중치는 아래와 같다.

```text
출고존 = 1.00
입고존 = 0.88
보관존 = 0.46
반품존 = 0.34
뮬랑 = 0.26
기타 = 0.40
```

### 2-4. 존 가중치 × 0.28

`0.28`은 존 중요도를 전체 우선이동점수에 28% 비중으로 반영하겠다는 뜻이다.  
즉, 출고존이나 입고존처럼 운영상 중요한 자리를 점유하고 있을수록 점수를 더 높게 부여한다.

예를 들어 입고존이면:

```text
0.88 × 0.28 = 0.2464
```

### 2-5. 해당 위치 총수량 ÷ 최대 위치수량

이 값은 현재 계산 중인 위치 수량이 전체 대상 위치 중 어느 정도 규모인지를 보는 비율이다.

- `해당 위치 총수량 = 가용수량 + 예약수량`
- `최대 위치수량 = 현재 필터에 걸린 전체 위치 중 가장 큰 수량`

예를 들어 해당 위치가 가용 3, 예약 0이면:

```text
해당 위치 총수량 = 3 + 0 = 3
```

전체 대상 위치 중 가장 큰 수량이 150이면:

```text
최대 위치수량 = 150
```

수량 비율은:

```text
3 ÷ 150 = 0.02
```

### 2-6. × 0.20

`0.20`은 수량 규모를 전체 우선이동점수에 20% 비중으로 반영하겠다는 뜻이다.  
즉, 많이 쌓인 위치일수록 공간을 더 크게 점유한다고 보고 점수를 높게 준다.

예를 들어 수량 비율이 `0.02`이면:

```text
0.02 × 0.20 = 0.004
```

### 2-7. 가용수량 ÷ 해당 위치 총수량

이 값은 해당 위치 재고 중 실제로 바로 이동 가능한 수량 비율이다.  
가용재고가 많을수록 실제 조치가 쉽다고 보기 때문에 점수에 반영한다.

예를 들어 가용 3, 총수량 3이면:

```text
3 ÷ 3 = 1
```

### 2-8. × 0.10

`0.10`은 가용 비율을 전체 우선이동점수에 10% 비중으로 반영하겠다는 뜻이다.

예를 들어 가용 비율이 `1`이면:

```text
1 × 0.10 = 0.10
```

## 3. 우선이동점수 예시 계산

예를 들어 어떤 위치가 아래 조건이라고 가정한다.

- 미출고일수: `240일`
- 존: `입고존-A`
- 가용수량: `3`
- 예약수량: `0`
- 최대 위치수량: `150`

그러면 계산은 아래처럼 된다.

```text
미출고 항목
= (240 ÷ 365) × 0.42
= 0.6575 × 0.42
= 0.2762

존 항목
= 0.88 × 0.28
= 0.2464

수량 항목
= (3 ÷ 150) × 0.20
= 0.02 × 0.20
= 0.0040

가용 항목
= (3 ÷ 3) × 0.10
= 1 × 0.10
= 0.1000
```

최종 합계는:

```text
0.2762 + 0.2464 + 0.0040 + 0.1000 = 0.6266
0.6266 × 100 = 62.66
반올림 = 63점
```

즉, 이 위치의 우선이동점수는 `63점`이다.

## 4. 평균 리스크란?

평균 리스크는 한 존 안에 있는 위치들의 우선이동점수를 모두 더한 뒤, 위치 개수로 나눈 값이다.  
즉, 그 존 전체의 위험도를 위치별 점수 평균으로 본 값이다.

산식은 아래와 같다.

```text
평균 리스크 = 해당 존에 속한 위치별 우선이동점수의 합 ÷ 해당 존의 위치 수
```

일반식으로 쓰면:

```text
avgRisk = Σ priorityScore ÷ locationCount
```

예를 들어 `입고존-D`에 위치가 2개 있고 각 점수가 `51점`, `78점`이면:

```text
평균 리스크
= (51 + 78) ÷ 2
= 129 ÷ 2
= 64.5점
```

즉 평균 리스크는 `무엇에 무엇을 곱한 값`이 아니라, `위치별 우선이동점수를 모두 더해서 위치 수로 나눈 값`이다.

## 5. 우선이동 비중이란?

우선이동 비중은 해당 존 전체 수량 중에서 우선 이동 대상 수량이 얼마나 차지하는지를 보는 값이다.

산식은 아래와 같다.

```text
우선이동 비중 = 우선이동 대상 수량 ÷ 해당 존 전체 수량 × 100
```

여기서:

- `우선이동 대상 수량` = `priorityScore >= 60` 인 위치들의 수량 합
- `해당 존 전체 수량` = 해당 존에 속한 모든 위치 수량의 합

예를 들어 `입고존-D`의 위치 2개 수량이 각각 `18`, `31`이고, 이 중 `78점`인 위치만 우선이동 대상이면:

```text
우선이동 대상 수량 = 31
해당 존 전체 수량 = 18 + 31 = 49

우선이동 비중
= 31 ÷ 49 × 100
= 63.3%
```

## 6. 점수 구간 해석

현재 프로젝트에서 점수 구간은 아래처럼 해석한다.

```text
75점 이상 = 즉시 이동
60점 이상 = 우선 이동
45점 이상 = 모니터링
45점 미만 = 관찰
```

또한 실행 대기열은 `55점 이상` 위치를 대상으로 표시한다.

## 7. 한 문장으로 설명하면

우선이동점수는 미출고 경과 정도, 존의 중요도, 위치 수량 규모, 가용 재고 비율을 각각 비율값으로 만든 뒤, 정해진 가중치를 곱해 더해서 만든 점수다.

평균 리스크는 해당 존에 포함된 위치들의 우선이동점수를 모두 더한 후 위치 수로 나눈 평균값이다.

우선이동 비중은 해당 존 전체 수량 중에서 우선 이동 대상 수량이 차지하는 비율이다.
