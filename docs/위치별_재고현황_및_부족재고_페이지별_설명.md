# 위치별 재고현황 및 부족 재고 현황 페이지별 설명

## 기획 방향

- `목록 페이지`: 상세 데이터 확인과 실무 처리
- `시각화 페이지`: 전체 현황 파악과 빠른 판단

즉, 목록은 실행용, 시각화는 판단용 화면입니다.

## 1. 위치별 재고현황

### 목록 페이지

- 기획 의도: 품목별 재고가 어느 위치에 얼마나 있는지 정확히 확인하기 위한 화면
- 핵심 목적: 위치, 수량, 가용재고를 기준으로 피킹, 이동, 보충 업무를 빠르게 판단
- 기대 효과: 재고 탐색 시간 단축, 작업 정확도 향상

### 시각화 페이지

- 기획 의도: 창고 내 재고 분포를 직관적으로 보여주기 위한 화면
- 핵심 목적: 재고 집중 구역, 분산 상태, 재배치 필요 구간을 한눈에 파악
- 기대 효과: 창고 가시성 향상, 재배치 판단 속도 향상

## 2. 부족 재고 현황

### 목록 페이지

- 기획 의도: 부족 상태이거나 최소 재고 이하 품목을 빠르게 확인하기 위한 화면
- 핵심 목적: 부족 품목, 부족 수량, 우선 대응 대상을 명확히 식별
- 기대 효과: 긴급 발주 및 보충 대응 속도 향상, 출고 리스크 감소

### 시각화 페이지

- 기획 의도: 부족 재고의 위험도와 우선순위를 직관적으로 보여주기 위한 화면
- 핵심 목적: 전체 부족 규모와 위험 구간을 한눈에 파악
- 기대 효과: 관리자 의사결정 지원, 선제 대응 강화

## 발표용 한 줄 정리

`위치별 재고현황`은 재고의 위치와 분포를 정확하게 파악하기 위한 기능이고,  
`부족 재고 현황`은 부족 위험을 빠르게 발견하고 대응하기 위한 기능입니다.

또한 `목록 페이지`는 상세 확인과 실행을 지원하고,  
`시각화 페이지`는 전체 현황과 우선순위 판단을 지원하도록 기획했습니다.
